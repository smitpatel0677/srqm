   512 ## 15. PWA & Offline Support
   513 
   514 SQRM is a full **Progressive Web App** (PWA).
   515 
   516 ### Features
   517 - **Add to Home Screen** — prompt shown to mobile users (dismissible, stored in `localStorage`)
   518 - **Service Worker** (`/sw.js`) — network-first strategy with cache fallback
   519 - **Offline banner** — yellow banner shown when `navigator.onLine` is false; disappears on reconnect
   520 - **Offline page** (`/offline.html`) — shown when navigation request fails with no cache
   521 
   522 ### Cache Strategy
   523 - Static assets cached on install (`style.css`, `app.js`, `offline.html`)
   524 - Dynamic pages cached on first visit (network-first)
   525 - API endpoints (`/api/*`) are never cached
   526 
   527 ### To update the service worker cache version:
   528 In `public/sw.js`, increment the cache names:
   529 ```js
   530 const CACHE_NAME   = 'sqrm-v2';   // bump this
   531 const STATIC_CACHE = 'sqrm-static-v2';
   532 ```

   536 ## 16. Order Flow
   537 
   538 ```
   539 Customer at table
   540     │
   541     ▼
   542 Scans QR code ──► /m/{slug} (Menu Page)
   543     │
   544     ▼
   545 Adds items to cart (stored in localStorage)
   546     │
   547     ▼
   548 /cart — enters Name, Phone, Payment Method
   549     │         (Online shows owner payment info; no gateway)
   550     ▼
   551 POST /api/order/place
   552     │
   553     ▼
   554 Redirect to /order/{unique_token}
   555     │
   556     ▼ (polls /api/order/status every 3s)
   557 Status: PENDING
   558     │
   559     ▼ (Owner dashboard polls /api/orders/pending every 4s)
   560 Owner hears ding.mp3 (new order sound)
   561     │
   562    [Owner clicks Accept]          [Owner clicks Reject]
   563     │                                     │
   564     ▼                                     ▼
   565 Status: ACCEPTED                   Owner enters reason
   566     │                              Status: REJECTED
   567     │                              Customer sees reason
   568     │                              + restaurant phone number
   569     │
   570    [Owner clicks Given/Completed]
   571     │
   572     ▼
   573 Status: COMPLETED
   574 PDF bill available for download (client-side jsPDF)
   575 Review popup shown to customer (optional)
   576 ```
   577 
   578 ### Order Status Values
   579 | Status | Meaning |
   580 |--------|---------|
   581 | `pending` | Submitted, waiting for owner |
   582 | `accepted` | Owner accepted the order |
   583 | `rejected` | Owner rejected (with reason) |
   584 | `completed` | Order delivered, bill generated |
